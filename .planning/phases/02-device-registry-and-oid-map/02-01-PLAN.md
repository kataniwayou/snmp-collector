Create five options classes in `src/SnmpCollector/Configuration/`. All use file-scoped namespace `SnmpCollector.Configuration`, are `public sealed` classes.

**DeviceOptions.cs** -- Copy from `src/Simetra/Configuration/DeviceOptions.cs` and adapt:
- Keep: `string Name`, `string IpAddress`, `List<MetricPollOptions> MetricPolls` (all with `= string.Empty` / `= []` defaults)
- REMOVE: `string DeviceType` entirely (no device module system in SnmpCollector)
- ADD: `string? CommunityString { get; set; }` (nullable, optional per-device override -- falls back to SnmpListenerOptions.CommunityString when null/empty)
- XML doc: Name is "Human-readable device name used as 'agent' label and Quartz job identity component", IpAddress is "IP address for SNMP polling and trap source matching", CommunityString is "Optional per-device SNMP community string override"

**MetricPollOptions.cs** -- REWRITE (not a copy from Simetra -- Simetra's version has MetricName, MetricType, OidEntryOptions which SnmpCollector does not use):
- `public List<string> Oids { get; set; } = [];` -- OIDs to fetch in this poll group, each resolves through OidMap
- `public int IntervalSeconds { get; set; }` -- polling interval, must be > 0
- NO `MetricName`, NO `MetricType`, NO `OidEntryOptions`, NO `Source`, NO `StaticLabels`
- XML doc on class: "One poll group for a device. Quartz job identity: metric-poll-{deviceName}-{pollIndex}"

**DevicesOptions.cs** -- Copy from `src/Simetra/Configuration/DevicesOptions.cs` with namespace change to `SnmpCollector.Configuration`:
- `public const string SectionName = "Devices";`
- `public List<DeviceOptions> Devices { get; set; } = [];`
- XML doc: "Wrapper for the Devices configuration array. Bound from 'Devices' section."

**OidMapOptions.cs** -- NEW (no Simetra equivalent):
- `public const string SectionName = "OidMap";`
- `public Dictionary<string, string> Entries { get; set; } = [];`
- XML doc: "Flat OID-to-metric-name mapping. Hot-reloadable via IOptionsMonitor (MAP-05). Maps OID strings to camelCase metric names."

**SnmpListenerOptions.cs** -- Copy from `src/Simetra/Configuration/SnmpListenerOptions.cs` with namespace change:
- Keep all four properties with their data annotations: `[Required] BindAddress`, `[Range(1, 65535)] Port`, `[Required] CommunityString`, `[Required][RegularExpression] Version`
- Keep `public const string SectionName = "SnmpListener";`
- Update XML doc on CommunityString: "Global default SNMP community string. Per-device overrides in DeviceOptions.CommunityString."

**DevicesOptionsValidator.cs** -- Copy from `src/Simetra/Configuration/Validators/DevicesOptionsValidator.cs` and adapt:
- Keep the overall structure: `Validate` method, `ValidateDevice`, `ValidateNoDuplicates`, `ValidateMetricPoll`
- REMOVE: `KnownDeviceTypes` HashSet and all `DeviceType` validation (no DeviceType in SnmpCollector)
- REMOVE: `ValidateOidEntry` method (no OidEntryOptions in SnmpCollector)
- KEEP: `ValidateDevice` checks:
  - `string.IsNullOrWhiteSpace(device.Name)` -> `"Devices[{index}].Name is required"`
  - `string.IsNullOrWhiteSpace(device.IpAddress)` -> `"Devices[{index}].IpAddress is required"`
  - ADD: else if `!System.Net.IPAddress.TryParse(device.IpAddress, out _)` -> `"Devices[{index}].IpAddress '{device.IpAddress}' is not a valid IP address"`
  - Loop over MetricPolls calling `ValidateMetricPoll`
- KEEP: `ValidateNoDuplicates` -- identical to Simetra (name + IP duplicate detection with HashSet)
- ADAPT: `ValidateMetricPoll` to validate SnmpCollector's simplified MetricPollOptions:
  - `poll.IntervalSeconds <= 0` -> `"{prefix}.IntervalSeconds must be greater than 0"`
  - `poll.Oids.Count == 0` -> `"{prefix}.Oids must contain at least one OID"`
  - Prefix format: `"Devices[{deviceIndex}].MetricPolls[{pollIndex}]"`
- Error message format follows Phase 1 pattern: "Section.Field is required" or "Section.Field condition"

  - `Version != "v2c"` -> `"SnmpListener:Version must be 'v2c'"`
  - `Port < 1 || Port > 65535` -> `"SnmpListener:Port must be between 1 and 65535"`
- Both use `using Microsoft.Extensions.Options;`
