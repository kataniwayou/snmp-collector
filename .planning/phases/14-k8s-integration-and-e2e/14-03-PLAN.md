  <action>
Create `deploy/k8s/verify-e2e.sh` with the following structure:

**Header and configuration:**
- `#!/usr/bin/env bash` with `set -euo pipefail`
- Variables: `PROMETHEUS_URL` (default `http://localhost:9090`), `POLL_INTERVAL=15`, `TRAP_TIMEOUT=300` (5 min), `POLL_TIMEOUT=60`
- Accept optional `--prometheus-url` argument to override default
- Require `curl` and `jq` as dependencies (check at start, fail with clear message if missing)

**Helper functions:**
- `check_metric(query, description)` -- queries Prometheus `/api/v1/query` endpoint, URL-encodes the query, returns 0 if result count > 0, 1 otherwise. Prints PASS/FAIL with series count.
- `wait_for_metric(query, description, timeout)` -- polls `check_metric` every `$POLL_INTERVAL` seconds until success or timeout. Prints TIMEOUT on failure.
- `start_port_forward()` -- starts `kubectl port-forward -n simetra svc/prometheus 9090:9090` in background, saves PID, waits 3 seconds for it to be ready. Uses trap to clean up on exit.
- `cleanup()` -- kills port-forward process if running.

**Poll metric checks (immediate, short timeout):**
These should appear within ~10-20 seconds of the collector running. Use `wait_for_metric` with `$POLL_TIMEOUT`.

1. OBP polled metrics exist: `snmp_gauge{device_name="OBP-01",source="poll"}`
2. OBP specific metric with name: `snmp_gauge{metric_name="obp_r1_power_L1",device_name="OBP-01"}`
3. NPB polled metrics exist: `snmp_gauge{device_name="NPB-01",source="poll"}`
4. NPB specific metric with name: `snmp_gauge{metric_name="npb_cpu_util",device_name="NPB-01"}`

**Trap metric checks (longer wait):**
Traps fire at random intervals (60-300s). Use `wait_for_metric` with `$TRAP_TIMEOUT`.

5. OBP trap metric: `snmp_gauge{metric_name=~"obp_channel_L.*",source="trap",device_name="OBP-01"}`
6. NPB trap metric: `snmp_gauge{metric_name=~"npb_port_status_P.*",source="trap",device_name="NPB-01"}`

**Summary:**
- Track pass/fail counts
- Print summary table at end
- Exit 0 if all pass, exit 1 if any fail

**URL encoding:** Use `curl --data-urlencode` with `-G` flag for GET with encoded query params instead of manual URL encoding. This avoids the python3 dependency:
```bash
curl -s -G "$PROMETHEUS_URL/api/v1/query" --data-urlencode "query=$query"
```

This script implicitly verifies simulator reachability: if polled metrics appear in Prometheus, the simulators are reachable. No explicit reachability check is needed.
- Prometheus metric name: The OTel gauge instrument `snmp_gauge` appears as `snmp_gauge` in Prometheus (no suffix transformation for gauges via remote write).
- The script assumes the cluster is already deployed and running. It does NOT apply any K8s manifests.
- Make the script executable: `chmod +x deploy/k8s/verify-e2e.sh`
