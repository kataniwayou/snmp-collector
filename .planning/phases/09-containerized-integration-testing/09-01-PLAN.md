<objective>
Fix the existing monitoring stack (OTel Collector + Prometheus) to use the push-based prometheusremotewrite pipeline instead of the scrape-based prometheus exporter.

Purpose: The existing K8s monitoring manifests use a scrape-based OTel Collector → Prometheus pipeline (OTel Collector exposes :8889, Prometheus scrapes it). SnmpCollector requires the push-based prometheusremotewrite pipeline (decision [01-02]). Without this fix, no metrics from any OTLP source reach Prometheus.

Output: Updated monitoring stack manifests that accept OTLP metrics via gRPC and push them to Prometheus via remote_write.
</objective>
