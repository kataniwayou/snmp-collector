  <action>
Create `src/SnmpCollector/config/oidmap-npb.json` following the exact structure of `oidmap-obp.json`.

**File structure:**
1. Header comment block (see RESEARCH.md Pattern 5 for exact template)
2. `"OidMap": { ... }` wrapper (CRITICAL -- without this, config binding silently produces empty dictionary)
3. System metrics section (4 entries)
4. Per-port sections for Ports 1-8 (64 entries total)

**System OIDs** (subtree `1.3.6.1.4.1.47477.100.1.{metricId}.0`):

| metricId | OID | Metric Name | SNMP Type | Comment |
|----------|-----|-------------|-----------|---------|
| 1 | `1.3.6.1.4.1.47477.100.1.1.0` | `npb_cpu_util` | OctetString | Units: load average (dimensionless), Range: 0.0-4.0 |
| 2 | `1.3.6.1.4.1.47477.100.1.2.0` | `npb_mem_util` | OctetString | Units: bytes (memory used), Range: 0-8589934592 |
| 3 | `1.3.6.1.4.1.47477.100.1.3.0` | `npb_sys_temp` | OctetString | Units: degrees Celsius, Range: 0.0-95.0 |
| 4 | `1.3.6.1.4.1.47477.100.1.4.0` | `npb_uptime` | OctetString | Units: human-readable duration, Range: N/A |

**Per-port OIDs** (subtree `1.3.6.1.4.1.47477.100.2.{portNum}.{metricId}.0`):

For each port 1-8, create 8 entries:

| metricId | Metric Pattern | SNMP Type | Comment |
|----------|---------------|-----------|---------|
| 1 | `npb_port_status_P{n}` | INTEGER | Values: 1=up, 2=down, 3=testing, Range: 1-3 |
| 2 | `npb_port_rx_octets_P{n}` | Counter64 | Units: bytes, Range: 0-2^64 (monotonic counter) |
| 3 | `npb_port_tx_octets_P{n}` | Counter64 | Units: bytes, Range: 0-2^64 (monotonic counter) |
| 4 | `npb_port_rx_packets_P{n}` | Counter64 | Units: packets, Range: 0-2^64 (monotonic counter) |
| 5 | `npb_port_tx_packets_P{n}` | Counter64 | Units: packets, Range: 0-2^64 (monotonic counter) |
| 6 | `npb_port_rx_errors_P{n}` | Counter64 | Units: packets, Range: 0-2^64 (monotonic counter) |
| 7 | `npb_port_tx_errors_P{n}` | Counter64 | Units: packets, Range: 0-2^64 (monotonic counter) |
| 8 | `npb_port_rx_drops_P{n}` | Counter64 | Units: packets, Range: 0-2^64 (monotonic counter) |

**JSONC comment format** (one line above each entry, matching OBP style):
```
// SNMP type: {type} | Units: {units} | Range: {range}
```
For port_status use `Values:` instead of `Units:`:
```
// SNMP type: INTEGER | Values: 1=up, 2=down, 3=testing | Range: 1-3
```

**Section separators** (matching OBP style):
- `// ==== System Metrics ====` before system entries
- `// ---- Port N ----` before each port's entries

**Critical rules:**
- NO trailing comma on the last entry (`npb_port_rx_drops_P8`)
- Port suffix is uppercase P: `_P1` not `_p1`
- Use underscore separation in metric names: `rx_octets` not `rxoctets`
- Every entry gets its own JSONC comment line above it
