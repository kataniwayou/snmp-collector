    Use the operations dashboard (simetra-operations.json) as the structural template. Clone the same skeleton patterns.

    **Dashboard skeleton:**
    - __inputs: DS_PROMETHEUS datasource placeholder (pluginId: prometheus)
    - __requires: grafana 11.0.0, prometheus 1.0.0, table panel
    - title: "Simetra Business"
    - uid: "simetra-business"
    - tags: ["simetra", "business"]
    - schemaVersion: 39
    - refresh: "5s"
    - time: { from: "now-15m", to: "now" }
    - graphTooltip: 1 (shared crosshair)
    - editable: true, timezone: "browser"
    - id: null

    **Template variable (device filter):**
    - name: "device", label: "Device", type: "query"
    - definition/query: label_values(snmp_gauge, device_name)
    - multi: true, includeAll: true, allValue: ".*"
    - current: { selected: true, text: "All", value: "$__all" }
    - refresh: 2 (on time range change)
    - datasource: ${DS_PROMETHEUS}

    **Panel layout (24-column grid, 4 panels total):**
    All panel IDs: null (Grafana auto-assigns on import).

    Row 1 - Gauge Metrics (y=0):
    - Row header panel: "Gauge Metrics" at y=0 (type: "row", collapsed: false, id: null)
    - Table panel at y=1, x=0, w=24, h=10
      - Query: snmp_gauge{device_name=~"$device"}
      - instant: true, format: "table"
      - datasource: ${DS_PROMETHEUS}
      - Field overrides to HIDE these columns: Time, __name__, ip, source, job, instance
      - Field overrides to SHOW and ORDER these columns:
        1. service_instance_id (rename to "Service Instance")
        2. device_name (rename to "Device")
        3. metric_name (rename to "Metric")
        4. oid (rename to "OID")
        5. snmp_type (rename to "SNMP Type")
        6. Value #A — keep visible, plain display, no formatting/thresholds/auto-units
      - No value mappings, no color thresholds — plain raw values

    Row 2 - Info Metrics (y=11):
    - Row header panel: "Info Metrics" at y=11 (type: "row", collapsed: false, id: null)
    - Table panel at y=12, x=0, w=24, h=10
      - Query: snmp_info{device_name=~"$device"}
      - instant: true, format: "table"
      - datasource: ${DS_PROMETHEUS}
      - Field overrides to HIDE these columns: Time, __name__, ip, source, job, instance, Value #A (the numeric 1.0 — meaningless for info metrics)
      - Field overrides to SHOW and ORDER these columns:
        1. service_instance_id (rename to "Service Instance")
        2. device_name (rename to "Device")
        3. metric_name (rename to "Metric")
        4. oid (rename to "OID")
        5. value (the LABEL column with actual string data, rename to "Value")
      - CRITICAL: The "value" label column contains the actual info data (e.g., "Linux server 5.4"). The "Value #A" column is always numeric 1.0 and must be hidden. These are DIFFERENT columns — "value" is a Prometheus label, "Value #A" is the metric value.

    **Column hiding pattern (from operations dashboard):**
    Use field overrides with matcher { id: "byName", options: "COLUMN_NAME" } and properties [{ id: "custom.hidden", value: true }].

    **Column renaming pattern (from operations dashboard):**
    Use field overrides with matcher { id: "byName", options: "COLUMN_NAME" } and properties [{ id: "displayName", value: "NEW_NAME" }].

    **Column ordering:**
    Use a Grafana "organize" transformation (transformations array) with indexByName to set column order, or use field overrides. Check the operations dashboard for the pattern used there and follow the same approach.

    **Critical anti-patterns to avoid:**
    - Do NOT hardcode datasource UIDs — always use ${DS_PROMETHEUS}
    - Do NOT use format "time_series" — must be "table" for labels to split into columns
    - Do NOT forget instant: true — without it you get hundreds of rows instead of one per device+metric
    - Do NOT assign panel IDs — use null for all
    - Do NOT confuse "value" (label) with "Value #A" (metric value) on the info table
    - Do NOT add color thresholds or unit formatting to the value column — plain display
    - Do NOT hardcode any device names anywhere

    **JSON formatting:** Pretty-print with 2-space indentation for readability.
