  <name>Task 1: Update K8s health probes with new community strings</name>
  <files>deploy/k8s/simulators/obp-deployment.yaml, deploy/k8s/simulators/npb-deployment.yaml</files>
  <action>
  Both deployment YAMLs have liveness and readiness probes that send raw SNMP GET packets with `public` (hex: `7075626c6963`, 6 bytes) hardcoded. These must be updated to use the correct community strings.

  **The raw SNMP packet approach:** The probes send a hand-crafted SNMP GET packet and check for any UDP response. The community string is embedded as an OCTET STRING in the packet. Changing the community changes:
  1. The community string bytes
  2. The OCTET STRING length byte (prefix)
  3. The SEQUENCE length bytes (outer packet wrapper)
  4. The inner message SEQUENCE length byte

  **Strategy:** Rather than hand-editing hex, rewrite the probe to use pysnmp (which is already installed in the container). This is simpler and more maintainable than recalculating ASN.1 length fields:

  ```python
  import sys
  from pysnmp.hlapi.v3arch.asyncio import *
  import asyncio

  async def check():
      engine = SnmpEngine()
      errorIndication, errorStatus, errorIndex, varBinds = await get_cmd(
          engine,
          CommunityData('Simetra.OBP-01'),
          await UdpTransportTarget.create(('127.0.0.1', 161), timeout=3, retries=0),
          ContextData(),
          ObjectType(ObjectIdentity('1.3.6.1.2.1.1.3.0'))
      )
      engine.close_dispatcher()
      sys.exit(1 if errorIndication else 0)

  asyncio.run(check())
  ```

  **For obp-deployment.yaml:**
  - Replace the inline hex-packet Python script in BOTH livenessProbe and readinessProbe with the pysnmp-based check using `Simetra.OBP-01` community
  - Add DEVICE_NAME env var with value "OBP-01"
  - Keep all existing env vars (TRAP_TARGET, TRAP_PORT, TRAP_INTERVAL_MIN, TRAP_INTERVAL_MAX)
  - Keep resource limits, timing parameters unchanged

  **For npb-deployment.yaml:**
  - Same probe replacement but with `Simetra.NPB-01` community
  - Add DEVICE_NAME env var with value "NPB-01"
  - Replace old env vars (TRAP_INTERVAL, STATE_CHANGE_INTERVAL) with TRAP_INTERVAL_MIN and TRAP_INTERVAL_MAX to match new simulator interface
  - Remove COUNTER_INCREMENT_MIN/MAX, UTIL_* env vars (no longer applicable -- new simulator uses hardcoded traffic profiles)
  - Keep TRAP_TARGET and TRAP_PORT

  **NOTE on the sysUpTime OID (1.3.6.1.2.1.1.3.0):** This is a standard MIB-II OID. pysnmp's engine responds to it automatically -- the simulator does NOT need to register it. If the query returns an error (e.g., because the simulator only serves enterprise OIDs), the probe should instead query one of the registered OIDs. Use `1.3.6.1.4.1.47477.10.21.1.3.1.0` for OBP (obp_link_state_L1) and `1.3.6.1.4.1.47477.100.1.1.0` for NPB (npb_cpu_util). These are guaranteed to exist.
