Create a new file `src/SnmpCollector/Jobs/MetricPollJob.cs` following RESEARCH.md Pattern 1 and the complete code example in the research.

**Class structure:**
- Namespace: `SnmpCollector.Jobs`
- `[DisallowConcurrentExecution]` attribute (locked decision: prevents pile-up on slow devices)
- `public sealed class MetricPollJob : IJob`
- Constant: `private const string SysUpTimeOid = "1.3.6.1.2.1.1.3.0";`

**Constructor DI dependencies (all singletons):**
- `IDeviceRegistry _deviceRegistry` — lookup device by name from JobDataMap
- `IDeviceUnreachabilityTracker _unreachabilityTracker` — record failure/success
- `ISender _sender` — MediatR dispatch (NOT IPublisher — locked decision from 03-06)
- `PipelineMetricService _pipelineMetrics` — IncrementPollExecuted, IncrementPollUnreachable, IncrementPollRecovered
- `ILogger<MetricPollJob> _logger`

**Execute method structure:**
1. Extract from `context.MergedJobDataMap`: `deviceName` (GetString!), `pollIndex` (GetInt), `intervalSeconds` (GetInt)
2. `var jobKey = context.JobDetail.Key.Name;`
3. Device lookup via `_deviceRegistry.TryGetDeviceByName(deviceName, out var device)`. On failure: LogWarning and `return` (do NOT enter the try block — snmp.poll.executed must NOT increment for config errors)
4. `var pollGroup = device.PollGroups[pollIndex];`
5. Build variable list: sysUpTime OID first via `new Variable(new ObjectIdentifier(SysUpTimeOid))`, then `pollGroup.Oids.Select(oid => new Variable(new ObjectIdentifier(oid)))`
6. Build endpoint: `new IPEndPoint(IPAddress.Parse(device.IpAddress), 161)` and community: `new OctetString(device.CommunityString)`
7. Try block:
   - Create linked CTS: `CancellationTokenSource.CreateLinkedTokenSource(context.CancellationToken)` with `CancelAfter(TimeSpan.FromSeconds(intervalSeconds * 0.8))` (locked decision: 80% of interval)
   - Call `Messenger.GetAsync(VersionCode.V2, endpoint, community, variables, timeoutCts.Token)`
   - Call `await DispatchResponseAsync(response, device, context.CancellationToken)`
   - On success: call `_unreachabilityTracker.RecordSuccess(deviceName)`. If returns true (transition), log Information "Device {Name} ({Ip}) recovered" and call `_pipelineMetrics.IncrementPollRecovered()`
8. Catch `OperationCanceledException` with guard `when (!context.CancellationToken.IsCancellationRequested)` — this is TIMEOUT (not shutdown). Log Warning "Poll job {JobKey} timed out waiting for SNMP response". Call `RecordFailure(deviceName, device)`.
9. Catch bare `OperationCanceledException` — this is HOST SHUTDOWN. `throw;` to let Quartz handle it.
10. Catch `Exception ex` — network/SNMP error. Log Warning with exception. Call `RecordFailure(deviceName, device)`.
11. Finally block: `_pipelineMetrics.IncrementPollExecuted()` — always fires per SC#4.

**DispatchResponseAsync private method:**
- Parameters: `IList<Variable> response, DeviceInfo device, CancellationToken ct`
- `uint? sysUpTime = null;`
- Foreach `variable` in `response`:
  - If `variable.Id.ToString() == SysUpTimeOid && variable.Data.TypeCode == SnmpType.TimeTicks`: extract `sysUpTime = ((TimeTicks)variable.Data).ToUInt32()` — then fall through to also dispatch as a varbind
  - If `variable.Data.TypeCode is SnmpType.NoSuchObject or SnmpType.NoSuchInstance or SnmpType.EndOfMibView`: LogDebug "OID {Oid} returned {TypeCode} from {DeviceName} -- skipping" and `continue`
  - Build `SnmpOidReceived` with: Oid = variable.Id.ToString(), AgentIp = IPAddress.Parse(device.IpAddress), DeviceName = device.Name, Value = variable.Data, Source = SnmpSource.Poll, TypeCode = variable.Data.TypeCode, SysUpTimeCentiseconds = sysUpTime
  - `await _sender.Send(msg, ct);`

**RecordFailure private method:**
- Parameters: `string deviceName, DeviceInfo device`
- Call `_unreachabilityTracker.RecordFailure(deviceName)`. If returns true (transition to unreachable):
  - Get failure count: `_unreachabilityTracker.GetFailureCount(deviceName)`
  - Log Warning "Device {Name} ({Ip}) unreachable after {N} consecutive failures" (locked decision format)
  - Call `_pipelineMetrics.IncrementPollUnreachable()`

- sysUpTime itself gets SysUpTimeCentiseconds = null on its own message (it's first in the loop, sysUpTime variable hasn't been extracted yet when processing the sysUpTime varbind itself).
