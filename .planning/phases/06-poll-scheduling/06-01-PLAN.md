Create the interface and implementation for per-device consecutive failure tracking.

**IDeviceUnreachabilityTracker.cs** (new file in Pipeline namespace):
- `bool RecordFailure(string deviceName)` — increments failure count; returns true ONLY on the transition to unreachable (when count reaches threshold). Returns false on subsequent failures after already unreachable.
- `bool RecordSuccess(string deviceName)` — resets failure count to 0; returns true ONLY on the transition from unreachable to healthy. Returns false if device was already healthy.
- `int GetFailureCount(string deviceName)` — returns current consecutive failure count (0 if never seen).
- `bool IsUnreachable(string deviceName)` — returns true if device is currently in unreachable state.

**DeviceUnreachabilityTracker.cs** (new file in Pipeline namespace):
- Sealed class implementing `IDeviceUnreachabilityTracker`.
- Hardcoded `_threshold = 3` (locked decision from CONTEXT.md: 3 consecutive failures).
- Uses `ConcurrentDictionary<string, DeviceState>` with `StringComparer.OrdinalIgnoreCase` (device names are user-configured).
- Inner `private sealed class DeviceState` with:
  - `volatile int _count` with `Interlocked.Increment` in RecordFailure
  - `volatile bool _isUnreachable` for transition detection
  - `RecordFailure(int threshold)` returns true on first reach of threshold (transition)
  - `RecordSuccess()` uses `Interlocked.Exchange(ref _count, 0)`, returns true only if was unreachable (transition)
- `GetOrAdd` pattern for both RecordFailure and RecordSuccess.

Follow the RESEARCH.md Pattern 5 exactly for the implementation. The inner class avoids ConcurrentDictionary struct-update atomicity issues.
