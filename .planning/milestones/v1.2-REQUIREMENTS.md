## v1.2 Requirements

### Configuration Management

- [x] **CFG-01**: Single ConfigMap key with all OID maps + device entries + JSONC documentation (replaces separate oidmap-*.json and devices.json)
  - *Outcome:* Implemented as split ConfigMaps (simetra-oidmaps + simetra-devices) with JSONC support. Split approach preferred for independent watcher lifecycle.
- [x] **CFG-02**: K8s API watch detects ConfigMap changes and reloads config at runtime (replaces file-based IOptionsMonitor hot-reload)
  - *Outcome:* OidMapWatcherService and DeviceWatcherService independently watch their ConfigMaps via K8s API. Sub-second event delivery verified live.
- [x] **CFG-03**: Local development fallback loads config from file when not running in K8s cluster
  - *Outcome:* Program.cs IsInCluster() gate loads oidmaps.json and devices.json from disk. Same downstream code path as K8s watchers.

### Operational Enhancements

- [x] **OPS-01**: Hot-reloadable device configuration (add/remove devices, change OIDs/intervals without pod restart, Quartz jobs re-registered dynamically)
  - *Outcome:* DynamicPollScheduler.ReconcileAsync diffs Quartz jobs. All scenarios verified live (add/remove/reschedule/OID changes).
