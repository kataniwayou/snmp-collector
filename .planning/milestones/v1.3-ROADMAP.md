# Milestone v1.3: Grafana Dashboards

**Status:** SHIPPED 2026-03-09
**Phases:** 18-19
**Total Plans:** 2

## Overview

This milestone delivered two purpose-built Grafana dashboard JSON files for the SNMP monitoring system. Claude creates the dashboard JSON files; the user manually imports them into Grafana via the UI. Phase 18 builds the operations dashboard for pipeline health and pod observability. Phase 19 builds the business dashboard with device-agnostic metric tables.

## Phases

### Phase 18: Operations Dashboard

**Goal**: Operators can monitor pipeline health, pod roles, and .NET runtime metrics across all replicas from a single Grafana dashboard JSON file
**Depends on**: Nothing (first phase of v1.3)
**Requirements**: DASH-01, DASH-02, OPS-01, OPS-02, OPS-03, OPS-04
**Plans**: 1 plan

Plans:
- [x] 18-01-PLAN.md -- Delete stale files, create operations dashboard JSON (pod table, 11 pipeline panels, 6 runtime panels)

**Success Criteria:**
1. Operations dashboard JSON file exists in deploy/grafana/dashboards/ ready for manual Grafana import
2. Stale reference project files under deploy/grafana/ are deleted and replaced
3. Operator sees a table listing each pod's service_instance_id, pod name, and current leader/follower role
4. Operator sees time series graphs for all 11 pipeline counters broken down by pod
5. Operator sees time series graphs for .NET runtime metrics broken down by pod
6. Dashboard refreshes automatically at a configurable interval showing live data

**Details:**
- Created `deploy/grafana/dashboards/simetra-operations.json` (1891 lines)
- 20 panels: pod identity table, 11 pipeline counters, 6 runtime, 3 row headers
- All panels filtered by Host Name dropdown (service_instance_id)
- Tooltip descriptions on all non-row panels
- DS_PROMETHEUS datasource placeholder for import
- Auto-refresh 5s, default range 15m

### Phase 19: Business Dashboard

**Goal**: Users can view current SNMP gauge and info metric values for any device in dynamically-populated tables without hardcoded device names
**Depends on**: Nothing (can run in parallel with Phase 18)
**Requirements**: BIZ-01, BIZ-02, BIZ-03, BIZ-04
**Plans**: 1 plan

Plans:
- [x] 19-01-PLAN.md -- Create business dashboard JSON (gauge metrics table, info metrics table, device filter, auto-refresh)

**Success Criteria:**
1. Business dashboard JSON file exists in deploy/grafana/dashboards/ ready for manual Grafana import
2. User sees a gauge metrics table with columns: service_instance_id, device_name, metric_name, oid, snmp_type, and value
3. User sees an info metrics table with columns: service_instance_id, device_name, metric_name, oid, and value
4. Tables auto-refresh to show live current values without manual page reload
5. Adding a new device to the system automatically populates it in the tables (no dashboard edits needed)

**Details:**
- Created `deploy/grafana/dashboards/simetra-business.json`
- 4 panels: 2 row headers, gauge table, info table
- 3 cascading filters: Host -> Pod -> Device
- Gauge table: Host, Pod, Device, Metric, OID, SNMP Type, Value, Trend (delta arrows), PromQL
- Info table: Host, Pod, Device, Metric, OID, SNMP Type, Value, PromQL
- Trend column uses delta(snmp_gauge[30s]) with colored arrows (green/red/neutral)
- PromQL column shows copyable query strings with all filter labels
- Cell inspect enabled on PromQL columns for click-to-copy

## Milestone Summary

**Key Decisions:**
- Dashboard JSON files created by Claude, imported manually by user via Grafana UI (no K8s provisioning)
- Two separate dashboards: operations (pipeline health) and business (SNMP metrics)
- Trend column with delta arrows preferred over per-row cell coloring (Grafana limitation)
- PromQL column with label_replace+label_join for copyable query strings
- Backtick raw strings in PromQL to avoid JSON double-escaping

**Issues Resolved:**
- Grafana configFromData doesn't support per-row coloring (fundamental limitation)
- PromQL regex escaping issues with pipe delimiter (switched to tilde)
- __tmp label not stripped by Prometheus (hidden via Grafana override)
- Info metrics Value column showing numeric 1 instead of string value (filterFieldsByName)

**Issues Deferred:**
- Stale comments in deploy/k8s/production/grafana.yaml referencing deleted dashboard files

**Technical Debt Incurred:**
- Stale comments in grafana.yaml (non-blocking, documentation only)
