# Requirements: SNMP Monitoring System

**Defined:** 2026-03-04
**Core Value:** Every SNMP OID — from a trap or a poll — gets resolved, typed correctly, and pushed to Prometheus where it's queryable in Grafana within seconds.

## v1 Requirements

### SNMP Collection

- [x] **COLL-01**: Trap listener receives SNMPv2c traps on UDP 162 with community string authentication
- [x] **COLL-02**: Quartz-based poller executes SNMP GET (v2c) for configured OIDs per device
- [x] **COLL-03**: Each device has its own IP, OID list, and configurable poll intervals in appsettings
- [x] **COLL-04**: Quartz creates a MetricPollJob per device/poll combination
- [x] **COLL-05**: Quartz thread pool auto-scales to total job count
- [x] **COLL-06**: Poll timeout set to 80% of interval to leave response window
- [x] **COLL-07**: Both traps and polls publish the same `SnmpOidReceived` notification to MediatR

### OID Map

- [x] **MAP-01**: Flat `Dictionary<string, string>` in appsettings under `OidMap` section
- [x] **MAP-02**: Maps OID string to metric_name
- [x] **MAP-03**: OID in map resolves to metric_name; OID not in map resolves to "Unknown"
- [x] **MAP-04**: Shared by traps and polls — no device distinction
- [x] **MAP-05**: Hot-reloadable without app restart

### MediatR Pipeline

- [x] **PIPE-01**: MediatR v12.5.0 (MIT license) for event routing
- [x] **PIPE-02**: `SnmpOidReceived` carries OID, agent IP, value, source (poll/trap), SNMP TypeCode
- [x] **PIPE-03**: `LoggingBehavior` logs every OID received with agent and source
- [x] **PIPE-04**: `ExceptionBehavior` catches all failures, never crashes the pipeline
- [x] **PIPE-05**: `ValidationBehavior` rejects malformed OID or IP format
- [x] **PIPE-06**: `OidResolutionBehavior` resolves metric_name from OID map
- [x] **PIPE-07**: `OtelMetricHandler` records to correct instrument based on SNMP TypeCode
- [x] **PIPE-08**: Behavior registration order: Logging -> Exception -> Validation -> OidResolution
- [x] **PIPE-09**: Pipeline uses IRequest<Unit>/ISender.Send for behavior chain execution (originally TaskWhenAllPublisher, changed to request/response pattern — behaviors bypass INotification)

### Business Metrics

- [x] **METR-01**: `snmp_gauge` instrument for Integer32, Gauge32, TimeTicks, Counter32, Counter64 values (counters exported as raw gauges — Prometheus rate() handles delta)
- [x] **METR-02**: Counter types (Counter32, Counter64) routed through snmp_gauge as raw values (snmp_counter instrument removed — delta computation deferred to Prometheus)
- [x] **METR-03**: `snmp_info` instrument (gauge=1) for OctetString, IpAddress, OID with string in `value` label
- [x] **METR-04**: Common labels on all instruments: `host_name`, `pod_name`, `metric_name`, `oid`, `device_name`, `ip`, `source`, `snmp_type`
- [x] **METR-05**: `host_name` loaded from NODE_NAME env var (K8s spec.nodeName) with MachineName fallback
- [x] **METR-06**: Runtime metric type detection from SNMP TypeCode

### Counter Delta Engine

- [x] **DELT-01**: Cache stores last cumulative value per OID+agent — *REMOVED: counters exported as raw gauges*
- [x] **DELT-02**: Delta computation — *REMOVED: Prometheus rate() handles natively*
- [x] **DELT-03**: Counter reset detection — *REMOVED*
- [x] **DELT-04**: Counter32 wrap-around handling — *REMOVED*
- [x] **DELT-05**: First poll skip — *REMOVED*
- [x] **DELT-06**: sysUpTime for reboot detection — *REMOVED*

### Pipeline Metrics

- [x] **PMET-01**: `snmp.event.published` — events published to MediatR
- [x] **PMET-02**: `snmp.event.handled` — events successfully handled
- [x] **PMET-03**: `snmp.event.errors` — exceptions caught by ExceptionBehavior
- [x] **PMET-04**: `snmp.event.rejected` — events rejected by ValidationBehavior
- [x] **PMET-05**: `snmp.poll.executed` — Quartz poll jobs completed
- [x] **PMET-06**: `snmp.trap.received` — traps received on UDP 162
- [x] **PMET-07**: All pipeline metrics include `host_name` and `pod_name` labels
- [x] **PMET-08**: Pipeline metrics exported by all instances (not leader-gated)

### Logging & Telemetry

- [x] **LOG-01**: Structured logging with `ILogger` and named placeholders
- [x] **LOG-02**: Custom console formatter: `[timestamp] [level] [hostname|podName|role|correlationId] category message`
- [x] **LOG-03**: Console output toggleable via `Logging:EnableConsole` in appsettings
- [x] **LOG-04**: OTLP log export with enrichment processor (host_name, role, correlationId)
- [x] **LOG-05**: Correlation IDs: rotating global via CorrelationJob + per-operation AsyncLocal scoping
- [x] **LOG-06**: OTLP gRPC export for metrics and logs to OTel Collector
- [x] **LOG-07**: No traces — metrics and logs only

### Leader Election & HA

- [x] **HA-01**: `ILeaderElection` interface with `IsLeader` and `CurrentRole` properties
- [x] **HA-02**: `K8sLeaseElection` using Kubernetes Lease API (auto-detected via `IsInCluster()`)
- [x] **HA-03**: `AlwaysLeaderElection` for local development (non-K8s environments)
- [x] **HA-04**: All instances poll devices and receive traps (not leader-only)
- [x] **HA-05**: Only leader exports business metrics (snmp_gauge, snmp_info)
- [x] **HA-06**: Pipeline metrics + System.Runtime exported by all instances
- [x] **HA-07**: `MetricRoleGatedExporter` filters business meter at export time
- [x] **HA-08**: Near-instant failover via explicit lease deletion on graceful shutdown

### Graceful Shutdown

- [x] **SHUT-01**: GracefulShutdownService registered last in DI (stops first)
- [x] **SHUT-02**: Step 1: Release K8s lease (3s budget)
- [x] **SHUT-03**: Step 2: Stop SNMP trap listener (3s budget)
- [x] **SHUT-04**: Step 3: Scheduler standby (3s budget)
- [x] **SHUT-05**: Step 4: Drain in-flight operations (8s budget)
- [x] **SHUT-06**: Step 5: Flush telemetry with independent CTS (5s budget)
- [x] **SHUT-07**: Each step has its own CancellationTokenSource budget
- [x] **SHUT-08**: Total shutdown timeout: 30 seconds

### Health Probes

- [x] **HLTH-01**: Startup probe: verify OID map loaded and poll definitions registered
- [x] **HLTH-02**: Readiness probe: verify trap listener running and registry populated
- [x] **HLTH-03**: Liveness probe: per-job staleness detection
- [x] **HLTH-04**: Job interval registry built at startup
- [x] **HLTH-05**: Liveness vector stamped by every job in finally block

### Hardening

- [x] **HARD-01**: Trap storm protection — rate limiting / backpressure
- [x] **HARD-02**: Device unreachability handling — timeout detection
- [x] **HARD-03**: SNMP poll timeout logged at Warning level
- [x] **HARD-04**: Configuration validation at startup with fail-fast

### Push Pipeline

- [x] **PUSH-01**: Full push: App -> OTLP gRPC -> OTel Collector -> remote_write -> Prometheus
- [x] **PUSH-02**: OTel Collector configuration for prometheusremotewriteexporter
- [x] **PUSH-03**: No scrape endpoint — pure push

### Device Configuration

- [x] **DEVC-01**: Per-device configuration with Name, IpAddress, and MetricPolls
- [x] **DEVC-02**: Each MetricPoll entry has an OID list and IntervalSeconds
- [x] **DEVC-03**: Device registry with O(1) lookup by IP and by name
- [x] **DEVC-04**: Quartz job identity: `metric-poll-{deviceName}-{pollIndex}`

## Milestone Summary

**Shipped:** 80 of 80 v1 requirements (6 removed as deliberate design change, 5 adjusted to match evolved implementation)

**Adjusted:**
- METR-02: snmp_counter instrument removed; Counter32/Counter64 exported as raw gauges via snmp_gauge
- METR-04: Labels changed from site_name/agent to host_name/pod_name/device_name/ip/snmp_type
- METR-05: site_name replaced with host_name from NODE_NAME env var
- PMET-07: site_name label replaced with host_name + pod_name
- PIPE-09: TaskWhenAllPublisher replaced with IRequest<Unit>/ISender.Send pattern

**Removed:**
- DELT-01 through DELT-06: Counter delta engine deliberately removed; Prometheus rate() handles counter delta natively

**Dropped:** None
