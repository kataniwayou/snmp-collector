# Milestone v1.0: Foundation

**Status:** Shipped 2026-03-07
**Phases:** 1-10
**Total Plans:** 48

## Overview

This roadmap builds a K8s-native SNMP monitoring agent from infrastructure up to production-ready operation. The build order is dictated by two hard dependency constraints — nothing can resolve OIDs without the registry, and nothing can push metrics without the pipeline — and two correctness constraints — OTel cardinality must be locked before any instrument is created, and counter delta logic must be correct before any counter metrics reach Prometheus. Ten phases deliver one coherent, verifiable capability each: foundation, registries, pipeline, delta engine, trap ingestion, poll scheduling, leader-gated export, production hardening, containerized testing, and metrics redesign.

## Milestone Summary

**Key Decisions:**
- MediatR 12.5.0 (MIT) locked — no v13+ (RPL-1.5 license)
- SNMPv2c only — no v3 auth/USM
- All instances poll and receive traps — leader election gates export only
- Three instruments (snmp_gauge, snmp_info) — snmp_counter removed post-Phase 4
- Flat OID map (exact-match dictionary) — no prefix/pattern matching
- Community string convention (Simetra.*) for device identity
- host_name from NODE_NAME env var (physical K8s node), pod_name from HOSTNAME (pod identity)
- Heartbeat is internal infrastructure — pipeline metrics prove liveness, no metric export

**Issues Resolved:**
- INotification → IRequest<Unit> (MediatR behaviors bypass INotification entirely)
- Counter delta engine removed — Prometheus rate() handles counters natively
- Per-device channels → single shared ITrapChannel
- site_name → host_name (environment-derived, not config-derived)
- Heartbeat metric pollution eliminated (handler suppression)

**Technical Debt Accepted:**
- REQUIREMENTS.md text stale (snmp_counter references, old label names)
- IDeviceRegistry.TryGetDevice(IPAddress) orphaned (community string replaced IP lookup)
- PollSchedulerStartupService thread pool log off-by-one
